配置开发环境

  • 使用anconda 生成虚拟编译环境
  • 创建GitHub仓库
  • 使用GitHubDesktop更新项目

在上传本地更新后的文件前先在工程目录下设置.gitignore

在这个文件中写上要忽略的文件避免一同将不必要的文件一起上传

在使用git命令合并分支前先fork团队仓库

  • 修改完成后在Desktop中新建Branch

选择Branch菜单栏中 的new branch选项

为新分支取名。此处已经有feat/change-team-models 一般不直接使用main分支

  • 将要提交的文件commite to new Branch

  • 然后发出 Pull Request 请求 等待管理员合并分支

第一次PR在Branch中最后一项pull request

之后直接点击Fetch origin

  • 当Pull Request通过,右键选中并Delet新建的Branch

构建ORM框架

  • 使用ponyorm创建Object Relational Mapping

一个框表示一个对象

对象可以有许多可编辑属性(字段)

两个对象间的线表示关系

比如User与Team(Group)的关系是 多个用户对应多个team(Group)

User和Contribution的关系是 一个用户多个贡献

而GroupContest与Reward的关系是 一个GroupContest可能有0/1个奖项

  • 关于为什么在team和contest之间设置中间属性

事实上只要是多对多关系，数据库实现上必须要开第三个表来连接

在orm里如果这个关系没有其他字段(属性)可以隐藏

但是我们的需求里需要在这个关系里加入证书之类的

数据库实现上必须要开第三个表来连接

  • user App包含:Training(训练记录), Region(擅长领域), User(注册用户), Contribution(贡献)
  • team App包含: Group(队伍名), GroupContest(过渡对象), Contest(比赛), Reward(获奖)
